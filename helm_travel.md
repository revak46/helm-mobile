## HELM OUTBOX — YOUR MOBILE EMAIL GATE

This is how Helm sends you intelligence while you're traveling. The system is GitHub-gated — nothing sends without your explicit approval.

### How It Works
1. **8am daily:** Mac Mini generates a digest email and creates a GitHub issue in `revak46/helm-outbox` labeled `draft`
2. **You review on phone:** Open GitHub mobile → `helm-outbox` → read the draft
3. **You approve:** Change the label from `draft` to `approved`
4. **Within 15 min:** Poller picks up the approved issue, sends the email to akembi20@gmail.com, closes the issue, labels it `sent`

### Quick Reference
- **Repo:** github.com/revak46/helm-outbox (private)
- **Labels:** `draft` → `approved` → `sent` (or `rejected` / `send-failed`)
- **Recipient:** akembi20@gmail.com (hardcoded allowlist — Helm can only email you)
- **Gmail sender:** helm@akembi.com (OAuth with gmail.readonly + gmail.send)

### If Something Looks Wrong
- Label the issue `rejected` instead of `approved` — poller ignores it
- If the poller fails, the issue gets `send-failed` with an error comment
- No silent failures — every outcome is labeled and visible

### Manual Trigger (from Mac Mini terminal if needed)
```bash
# Force a new digest draft
python3 ~/Project_Atlas/helm-system/helm_outbox_draft.py

# Force a travel update draft
python3 ~/Project_Atlas/helm-system/helm_outbox_draft.py travel

# Force the poller to check now
python3 ~/Project_Atlas/helm-system/helm_outbox_send.py
```
